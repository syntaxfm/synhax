## Drizzle-Zero Integration

This project uses **drizzle-zero** to generate Zero schemas from Drizzle ORM
schemas. See: https://github.com/rocicorp/drizzle-zero

### How It Works

1. Define your database schema in Drizzle (`src/db/schema.ts`)
2. Run `drizzle-zero generate` to generate the Zero schema
   (`src/sync/zero-schema.gen.ts`)
3. The generated schema is imported and used in `src/sync/schema.ts`

### Generation Command

```bash
pnpm drizzle-zero generate --format
```

### Key Points

- **Never manually edit `zero-schema.gen.ts`** - it's auto-generated
- Column names use JavaScript field names (e.g., `authorId`), not DB column
  names (`author_id`)
- Relationships defined in Drizzle are automatically converted to Zero
  relationships
- Custom types with `$type<>()` are passed through to Zero

### Drizzle-Zero Config (optional)

Create `drizzle-zero.config.ts` to customize which tables/columns sync:

```ts
import { drizzleZeroConfig } from 'drizzle-zero';
import * as drizzleSchema from './src/db/schema';

export default drizzleZeroConfig(drizzleSchema, {
	tables: {
		users: {
			id: true,
			name: true,
			email: false // exclude from sync
		}
	}
});
```

## Zero 0.25+ Patterns

### Custom Mutators (Required in 0.25+)

CRUD mutators are deprecated. Always use custom mutators:

```ts
import { createMutators } from '@rocicorp/zero';

export const mutators = createMutators(schema, {
	todo: {
		insert: (tx, todo) => tx.todo.insert(todo),
		update: (tx, todo) => tx.todo.update(todo),
		delete: (tx, { id }) => tx.todo.delete({ id })
	}
});
```

### Synced Queries with defineQueries (Required in 0.25+)

Ad-hoc queries are deprecated. Use defineQueries for reusable, validated
queries:

```ts
import { createBuilder, defineQueries, defineQuery } from '@rocicorp/zero';
import { type } from 'arktype'; // or zod for validation

export const zql = createBuilder(schema);

export const queries = defineQueries({
	todo: {
		all: defineQuery(() => zql.todo),
		byCompleted: defineQuery(
			type({ completed: 'boolean' }),
			({ args: { completed } }) => zql.todo.where('completed', '=', completed)
		),
		byId: defineQuery(type({ id: 'string' }), ({ args: { id } }) =>
			zql.todo.where('id', '=', id).one()
		)
	}
});
```

### Permissions (0.25+ Pattern)

Define permissions in the schema file:

```ts
import { definePermissions } from '@rocicorp/zero';

export const permissions = definePermissions<AuthData, Schema>(
	schema,
	(allow) => ({
		todo: {
			row: {
				select: allow.if((authData, row) => row.userId === authData.sub),
				insert: allow.if((authData) => !!authData.sub),
				update: allow.if((authData, row) => row.userId === authData.sub),
				delete: allow.if((authData, row) => row.userId === authData.sub)
			}
		}
	})
);
```

## Key Rules

1. **Never use deprecated patterns**: No CRUD mutators, no ad-hoc queries, no
   RLS permissions
2. **Always validate query arguments**: Use arktype or zod with defineQuery
3. **Use createBuilder for raw ZQL**: When you need dynamic queries not covered
   by defineQueries
4. **Register DefaultTypes**: Add
   `declare module '@rocicorp/zero' { interface DefaultTypes { schema: Schema; } }`
   to avoid explicit generics
5. **Relationships**: Defined in Drizzle and auto-generated by drizzle-zero
6. **Never edit generated files**: `zero-schema.gen.ts` is auto-generated from
   Drizzle
